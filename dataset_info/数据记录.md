# MTCMB 数据记录（Data Record）

MTCMB（Multi-Task TCM Assessment Benchmark，多任务中医评估基准）是一个公开可用的数据集，用于系统评估大语言模型在中医领域的知识、推理与安全性。[Zenodo](https://doi.org/10.5281/zenodo.20465629)（CC-BY 4.0）仅发布 **12** 个子数据集、**7,100** 条样本的数据文件；[GitHub](https://github.com/Wayyuanyuan/MTCMB) 提供完整项目（含 `data/`、元数据、标准答案与评估脚本）。数据涵盖知识问答、语言理解、诊断推理、处方推荐与安全评估五个维度。

## 数据集格式与结构

每条数据均以 JSON 对象形式存储，采用 JSONL（JSON Lines）格式，每行代表一条完整的样本数据。根据任务类型的不同，数据结构包含以下标准化字段：

### 选择题类型（Multiple Choice Questions, MCQ）

适用于 TCM-ED-A、TCM-ED-B、TCM-SE-B 等数据集，包含以下字段：

- **id**：一个整数，用于唯一标识每道题目。
- **question**：一道以中文编写的问题题干。
- **options**：一个对象或数组，包含多个选项（通常为 A、B、C、D、E）。
- **answer**：该问题的标准答案（通常为选项字母，如 "A"、"B" 等）。

### 开放题类型（Open-Ended Questions）

适用于 TCM-FT、TCMeEE、TCM-CHGD、TCM-MSDD、TCM-Diagnosis、TCM-PR、TCM-FRD、TCM-SE-A 等数据集，包含以下字段：

- **id**：一个整数，用于唯一标识每道题目。
- **question**、**text**、**Medical_case**、**disease_case**、**dialogue** 等：依任务语义命名的输入字段。
- **answer** / **points**：标准答案；TCM-FT 评测对齐 `points`（要点），其余多为自然语言或结构化 JSON。
- **source**：样本溯源（所有子集均包含）。

### 阅读理解类型（Reading Comprehension）

适用于 TCM-LitData 数据集，包含以下字段：

- **id**：一个整数，用于唯一标识每条数据。
- **text**：一段文献文本内容。
- **annotations**：一个数组，包含多个问题-答案对，每个对象包含：
  - **Q**：基于文本内容的问题。
  - **A**：该问题的标准答案，为文本中的连续片段。

数据示例如下：

**示例 1：选择题类型（TCM-ED-A）**

```json
{
  "id": 0,
  "question": "与舌没有直接联系的脏是",
  "options": {
    "A": "肝",
    "B": "心",
    "C": "脾",
    "D": "肺",
    "E": "肾"
  },
  "answer": "A"
}
```

## 数据集概览与组成

该数据集共包含 7,100 道题目，分布在 12 个子数据集中，涵盖五个核心评估维度。以下介绍其主要特征。

### 按评估维度分布

题目涵盖五个逐级递进的能力维度：知识问答、语言理解、诊断推理、处方推荐和安全评估。如表 1 所示：

- **知识问答**维度包含 3 个数据集，共 6,100 条（占比 85.9%），旨在评估模型对中医基础理论、方剂学、针灸学、诊断学等核心知识的理解与应用能力。该维度数据量最大，反映了知识掌握是中医能力评估的基础。

- **语言理解**维度包含 3 个数据集，共 300 条（占比 4.2%），通过实体抽取、医案生成和文献问答等形式，评估模型在中医文本理解和信息抽取方面的表现力与准确性。

- **诊断推理**维度包含 2 个数据集，共 300 条（占比 4.2%），考察模型根据患者临床信息进行辨证分析，并准确判断疾病名称与证型的能力。

- **处方推荐**维度包含 2 个数据集，共 300 条（占比 4.2%），评估模型根据病情描述和证型特征推荐合适中药方剂的能力。

- **安全评估**维度包含 2 个数据集，共 100 条（占比 1.4%），通过填空题与选择题的形式，考察模型识别中医实践中涉及的安全风险的能力。

这种分布体现了在知识基础、应用能力和安全规范之间所做的有意平衡，既重视基础知识的掌握，也关注实际应用场景中的综合能力。

### 按任务类型分布

每道题目被标注为以下三种任务类型之一：选择题（MCQ）、开放题（Open-Ended）或阅读理解（Reading Comprehension）。如表 2 所示：

- **选择题**占 71.8%（5,100 条），主要分布在知识问答和安全评估维度，便于结构化评估和模型间比较，能够快速验证模型对特定知识点的掌握程度。

- **开放题**占比 26.8%（1,900 条），主要分布在语言理解、诊断推理和处方推荐维度，旨在评估模型的生成式推理能力和中医表达能力，不依赖预设选项，更能反映模型的真实能力水平。

- **阅读理解**占比 1.4%（100 条），仅出现在 TCM-LitData 数据集中，要求模型基于文献文本内容回答问题，评估模型的文本理解和信息提取能力。

这种分布体现了在评估便捷性与推理深度之间所做的有意平衡，既保证了大规模评估的效率，也确保了深度推理能力的充分测试。

### 按数据来源分布

数据集来源于多个权威渠道（与论文 Table 1 一致）：

- **国家考试题库**：TCM-ED-A（中级主治医师官方题库，12 学科各 100 题）、TCM-ED-B（执业医师官方题库，8 套模拟卷），共 6,000 条。

- **问答与文献**：TCM-FT 来自《中医学问答题库》；TCM-LitData 来自[阿里云天池中医文献问题生成数据集](https://tianchi.aliyun.com/dataset/dataDetail?dataId=86895)。

- **临床与教材**：TCMeEE 来自[中医智库](https://zhongyigen.com/)（95%）及执业医师提交医案（5%）；TCM-CHGD 来自国家中医药管理局认证中心实践技能案例分析题库；TCM-Diagnosis 与 TCM-FRD 来自国家“十三五”规划中医教材（内外妇儿各 50 例）。

- **竞赛数据**：TCM-MSDD、TCM-PR 来自 [CCL25-Eval 任务 9](https://tianchi.aliyun.com/competition/entrance/532301) 子任务 1 与 2。

- **药典安全记录**：TCM-SE-A、TCM-SE-B 基于《中华人民共和国药典》（2020 年版）毒性药材、配伍禁忌与妊娠禁忌，由 DeepSeek-R1 合成并经专家审核。

## 目录结构（Directory Structure）

与论文 Data Record 一致，仓库对外呈现两个主目录：

| 目录 | 说明 |
| ---- | ---- |
| **`data/`**（Primary Dataset） | **12** 个子数据集 JSONL，每行一条样本，含标准答案 `answer` 与溯源 `source` |
| **`dataset_info/`**（Metadata Documentation） | 各子集 Markdown 说明：来源、预处理、评估指标等 |

> **实现说明**：评测代码通过 `mtcmb_datasets.load_records(purpose=..., shot=...)` 从同一份 `data/` 派生零样本/少样本输入视图，无需再维护 `data_few_shot/` 等物理副本。  
> 历史上曾存在 `data/`、`data_few_shot/`、`evaluate/standard_answer/`、`evaluate/standard_answer_few_shot/` 四套副本，原因是两个正交维度被拆成物理文件夹：
>
> | 维度 | 说明 |
> |------|------|
> | **用途** | `data*` 给模型做题（`answer` 为空）；`standard_answer*` 供评测打分（`answer` 完整） |
> | **提示策略** | zero-shot 全量 id；few-shot 去掉提示词里已硬编码为示例的 id |
>
> 现仅保留 **`data/` 一份完整 JSONL**（含标准答案），通过根目录模块 `mtcmb_datasets.py` 的 `load_records(purpose=..., shot=...)` 按需派生视图。  
> 合并脚本：`python scripts/consolidate_data_to_canonical.py`

### data/ 目录（权威数据）

该目录为**唯一权威副本**，每条样本含完整标准答案。评测与生成时由 `mtcmb_datasets` 加载：

- `purpose=benchmark`：清空 `answer`（等同原 `data/` / `data_few_shot/` 的模型输入）
- `purpose=standard`：保留答案（等同原 `evaluate/standard_answer*'`）
- `shot=zero`：全量 id；`shot=few`：排除 few-shot 示例 id

命令行默认 `--step-chat data` 与 `--standard-answer-root data`，配合 `--prompt-type`（0=zero，1=few，2=CoT 与 zero 同题集）即可，**无需再切换目录**。

目录中包含 12 个 JSONL 格式的数据文件，分别对应 12 个子数据集：

- `1.TCM_ED_A.jsonl`：TCM-ED-A（知识问答，1,200 条）
- `2.TCM_ED_B.jsonl`：TCM-ED-B（知识问答，4,800 条）
- `3.TCM_FT.jsonl`：TCM-FT（知识问答，100 条）
- `4.TCMeEE.jsonl`：TCMeEE（语言理解，100 条）
- `5.TCM_CHGD.jsonl`：TCM-CHGD（语言理解，100 条）
- `6.TCM_LitData.jsonl`：TCM-LitData（语言理解，100 条）
- `7.TCM_MSDD.jsonl`：TCM-MSDD（诊断推理，100 条）
- `8.TCM_Diagnosis.jsonl`：TCM-Diagnosis（诊断推理，200 条）
- `9.TCM_PR.jsonl`：TCM-PR（处方推荐，100 条）
- `10.TCM_FRD.jsonl`：TCM-FRD（处方推荐，200 条）
- `11.TCM_SE_A.jsonl`：TCM-SE-A（安全评估，50 条）
- `12.TCM_SE_B.jsonl`：TCM-SE-B（安全评估，50 条）

## 评估注释（Evaluation Annotations）

每个数据集均配有标准的评估方法和评估指标，详细说明如何对模型输出进行评分。这些评估注释不仅提升了模型评估的标准化与可复现性，还可作为模型训练或提示（prompting）大语言模型时的参考标准。

评估指标包括：

- **准确率（Accuracy）**：适用于 TCM-ED-A、TCM-ED-B、TCM-SE-B（选项字母精确匹配）及 TCM-MSDD（证型/疾病多标签匹配，见 `TCM-MSDD.md`）。

- **BERTScore**：适用于开放题类型，通过语义相似度评估模型输出的质量。

- **ROUGE 和 BLEU**：适用于文本生成任务，评估模型输出的流畅性和准确性。

- **大模型评分**：TCM-SE-A 采用 GLM-4-Air-250414 对禁忌表述的临床适当性打分（非参评模型）。

- **集合匹配与综合分**：TCM-PR 采用 Jaccard、Precision、Recall、F1、Size Agreement 及实例综合分。

## 软件工具包（Software Package）

数据集附带一个基于 Python 的完整工具包，用于简化数据加载、模型调用和评估流程。该工具包提供以下功能：

- **数据读取和解析**：支持 JSONL 格式数据的批量加载和解析，自动识别不同任务类型的数据结构。

- **模型调用接口**：提供基于 OpenAI API 的远程调用模板，以及支持本地模型（HuatuoGPT-II、Taiyi-LLM、WiNGPT2）的调用接口，用户可轻松扩展支持其他模型。

- **评估框架**：包含完整的评估脚本，支持多种评估指标，可根据不同任务类型自动选择合适的评估方法。

- **并发处理**：支持多进程并发处理，可显著提升大规模评估的效率。

- **提示策略支持**：提供 zero-shot、few-shot 和 CoT（Chain of Thought）三种提示策略的示例数据和使用模板。

该工具包旨在帮助研究人员高效地使用该中医评估数据集，快速开展模型评估和对比实验。
